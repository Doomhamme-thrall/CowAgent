# Changelog

## v2.0.7 — 定时任务管理 CRUD 操作入口 (2026-05-19)

### 新增

- **WebUI 定时任务新增入口**：任务视图头部新增「新增任务」按钮，点击打开任务弹窗。
- **WebUI 定时任务编辑入口**：任务卡片上新增编辑按钮（✏️），点击弹窗中预填现有数据，支持修改名称、类型、消息内容、调度配置等。
- **WebUI 定时任务删除入口**：任务卡片上新增删除按钮（🗑️），点击弹出确认对话框，确认后删除任务。
- **WebUI 定时任务启停切换**：任务卡片上新增启停按钮（⏸/▶️），一键切换启用/停用状态，状态标签和图标即时变化。

### 变更

- **后端接口扩展**：`/api/scheduler` 从只读 GET 扩展为完整 REST 接口：
  - `POST /api/scheduler` — 创建任务
  - `PUT /api/scheduler` — 编辑任务 / 切换启用状态
  - `DELETE /api/scheduler` — 删除任务
  - `GET /api/scheduler` — 列表（保持不变）
- **SchedulerTool Agent 工具增强**：新增 `update` action，Agent 对话中也可编辑定时任务。
- **任务视图重构**：`loadTasksView()` 从只读列表升级为可操作卡片，每张卡片包含名称、状态标签、调度描述、下次执行时间、接收者以及编辑/启停/删除按钮。
- **i18n 补充**：新增中英文任务管理相关词条约 20 条。

### 技术细节

- 前后端使用统一的任务数据模型（`task_store.py` 的 `TaskStore`）
- 调度解析逻辑从 `SchedulerTool._parse_schedule` 抽取为独立函数 `_scheduler_parse_schedule`，供 Web 接口复用
- 任务创建/编辑均自动计算 `next_run_at`
- 删除操作使用与模型配置删除一致的 `showConfirmDialog` 确认对话框
