# Changelog

## v2.0.8 — 技能管理 CRUD 操作入口 (2026-05-19)

### 新增

- **WebUI 技能新增入口**：技能视图头部新增「新增技能」按钮，点击打开技能弹窗，填写名称、描述、Markdown 内容后保存即可创建自定义技能。
- **WebUI 技能编辑入口**：自定义技能卡片上新增编辑按钮（✏️），点击弹窗中预填现有数据（描述、Markdown 正文、启用状态），支持修改描述、Markdown 内容和启用开关。
- **WebUI 技能删除入口**：自定义技能卡片上新增删除按钮（🗑️），点击弹出确认对话框，确认后删除技能目录和配置。
- **WebUI 技能来源标识**：技能卡片展示来源标签——内置技能显示「内置」标签（仅可启停），自定义技能显示「自定义」标签（完整 CRUD）。

### 变更

- **后端接口扩展**：`/api/skills` 从只读 GET + open/close POST 扩展为完整 REST 接口：
  - `POST /api/skills`（action=create）— 创建技能
  - `PUT /api/skills` — 编辑技能（修改描述 / Markdown 内容 / 启用状态）
  - `DELETE /api/skills` — 删除技能目录
  - `GET /api/skills` — 列表（保持不变，新增 `content` 字段返回自定义技能的 Markdown 正文）
- **SkillManager 新增方法**：`create_skill()` / `update_skill()`，支持在 `workspace/skills/` 下创建和管理 SKILL.md 文件。
- **SkillService 新增方法**：`create()` / `update()`，供 WebUI 调用。
- **技能视图重构**：`loadSkillsSection()` 从简单列表升级为可操作卡片，每张卡片包含名称、启用/停用状态标签、来源标签（内置/自定义）、描述摘要以及编辑/启停/删除按钮。
- **i18n 补充**：新增中英文技能管理相关词条约 14 条。

### 技术细节

- 创建技能时自动在 `workspace/skills/{name}/` 目录下生成带 YAML frontmatter 的 `SKILL.md` 文件
- 编辑技能时同步更新 `skills_config.json` 和 `SKILL.md` 文件，确保 `refresh_skills()` 始终读到最新内容
- 内置技能（`source == "builtin"`）在 PUT/DELETE 时被拒绝，仅支持启用/停用
- 删除操作使用与定时任务一致的 `showConfirmDialog` 确认对话框
- 全部更改经过 30 项自动化测试验证
